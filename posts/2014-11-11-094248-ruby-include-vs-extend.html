簡單來說，module被include時，module的method會變成class的instance method，而被extend時，則會變成class method：

module M
  def say
    "hello"
  end
end

class A
  include(M)
end

a = A.new
a.say # "hello"
# A.say # undefined method `say' for A:Class (NoMethodError)

class B
  extend(M)
end

B.say # "hello"
# b = B.new
# b.say # undefined method `say' for # (NoMethodError)

如果要同時加入instance與class method，可以使用合體技：

module M
  def self.included(base)
    base.extend(ClassMethods)
  end

  module ClassMethods
    def say_hello
      "hello"
    end
  end

  def say_hi
    "hi"
  end
end

class A
  include(M)
end

A.say_hello # "hello"
a = A.new
a.say_hi # "hi"

rails的ActiveSupport::Concern則把上面的合體技更加的簡化：

require "active_support/concern"

module M
  extend ActiveSupport::Concern

  module ClassMethods
    def say_hello
      "hello"
    end
  end

  def say_hi
    "hi"
  end
end

class A
  include(M)
end

A.say_hello # "hello"
a = A.new
a.say_hi # "hi"

當然concern的功能不只這些，有興趣可以參考ActiveSupport::Concern的文件
